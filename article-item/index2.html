CSS - Tipografías

Hay muchos artículos y buenas prácticas establecidas en CSS que se basan en determinar cuáles son las unidades correctas o "mejores" para usar. Ahora que las funciones de comparación están bien soportadas en CSS, ya no tenemos que elegir.

Algo que quiero destacar de los últimos artículos de esta serie es que mis soluciones suelen implicar combinar y comparar unidades.

En el artículo sobre las preferencias de tamaño de fuente del usuario, señalé que los tamaños base en em multiplican la configuración del usuario. Si pedimos un tamaño de fuente de 20px usando 1.25em, y el usuario también pide 20px, el resultado no es un acuerdo, sino una combinación: 25px. Si usamos solo px, ignoramos por completo la preferencia del usuario, lo cual es peor.

Pero si dejamos de hacer los cálculos en nuestra cabeza y le damos al navegador ambas unidades, podemos hacer comparaciones mucho más interesantes y útiles: 


        html {
          /* usar el mayor de los dos */
          font-size: max(1em, 20px);

          /* limitar dentro de un rango cercano a la preferencia del usuario */
          font-size: clamp(1em, 20px, 1.25em);

          /* calcular el promedio */
          font-size: calc((1em + 24px) / 2);
        }

En los tres casos no se requiere conversión. Indicamos claramente el tamaño en px que buscamos y lo comparamos con el em del usuario. Las unidades tienen significados distintos, y esos significados son útiles para expresar mejor la negociación entre el sitio y el usuario.

Me gusta pensar las unidades relativas de CSS como variables provistas por el navegador, a las que les pasamos un multiplicador. Pensar así me obliga a enfocarme en el significado, no en el valor en px que asumimos.

Usar la unidad que signifique lo que querés decir

Una vez que resolví esa negociación en el elemento raíz, puedo referirme al resultado como 1rem. Para mí, 1rem siempre es un cálculo con una variable. De forma similar, 1em es el tamaño de fuente actual del contexto.

La diferencia entre 1rem y 1em es como la diferencia entre --brand-pink y currentColor. Ambos son variables, ambos son útiles, pero describen relaciones completamente distintas. Preguntar cuál es "mejor" no tiene sentido.

Cómo manejo el espaciado en CSS

Esta serie comenzó a partir de un artículo de Ashlee Boyer sobre usar px para el espaciado. Su punto es que los usuarios que hacen zoom quieren ampliar el contenido, y que ampliar el espacio al mismo ritmo puede afectar la legibilidad.

No tenemos que elegir entre px y em/rem como únicas opciones. Podemos describir mejor cómo pensamos el espacio en blanco, teniendo en cuenta tanto el tamaño de fuente como el espacio disponible.

Uso varias unidades de espaciado. Mi favorita es lh. Cuando separo párrafos o listas, quiero mantener un ritmo consistente. Si necesito tener en cuenta el espacio disponible, combino unidades y uso funciones matemáticas: 


        .card {
          --min: min(1lh, 2vi);
          --nearest-half: round(up, 2vi, 0.5lh);
        }

Si necesitás exactamente 12px, usá 12px. Pero si querés responder al tamaño de fuente y al espacio disponible, una combinación de unidades expresa mejor la intención.

No existe "la mejor unidad". Existen las unidades que expresan mejor lo que querés decir.
